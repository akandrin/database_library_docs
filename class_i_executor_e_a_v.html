Подробное описание

Интерфейс исполнителя запросов EAV.

Он предоставляет базовые запросы, которые могут оказаться полезными при работе с базой данных, представленной схемой EAV.

Правила именования таблиц для базы данных определяются интерфейсом IExecutorEAVNamingRules.

Метод SetRegisteredEntities может создать таблицы по указанным правилам, если передать в него флаг createTables = true. Остальные методы не создают таблицы, а пользуются существующими, созданными этим методом. При отсутствии таких таблиц команды будут выдавать ошибку.

Все методы класса не открывают и не закрывают транзакции. Таким образом, можно в вызывающем коде открыть транзакцию, вызвать один или несколько методов класса, после чего зафиксировать (или отменить) транзакцию, применив (или не применив) разом несколько изменений.

(Если вызывающий код не открывает транзакцию, то каждый метод класса будет работать в рамках одной или нескольких транзакций (зависит от реализации конкретного метода))

Получить значения всех атрибутов сущности. Для определения типов атрибутов метод использует GetRegisteredEntities. Таким образом, метод не вернет информацию о незарегистрированных типах атрибутов, если они есть.

Аргументы
entityNameНазвание сущности
entityIdИдентификатор сущности

Метод возвращает также те пары атрибут-значение, в которых значение не задано: тогда значение будет представлено пустой SQL-переменной. В случае успеха метод очищает вектор перед вставкой.

◆ InsertOrUpdate()

virtual IExecuteResultStatusPtr IExecutorEAV::InsertOrUpdate ( const EntityName & entityName,
EntityId entityId,
const AttrName & attrName,
const ValueType & value )
pure virtual

Обновить значение для атрибута сущности в таблице значений или вставить значение атрибута в таблицу значений, если такого значения ещё не было. А также вставить название атрибута в таблицу атрибутов, если его там ещё не было.

Аргументы
entityNameНазвание сущности
entityIdИдентфикатор сущности. При передаче несуществующего идентификатора вернется ошибка.
attrNameНаименование атрибута. При передаче названия несуществующего атрибута будет добавлен новый атрибут с таким названием.
valueЗначение атрибута. Это обязательно должна быть ненулевая переменная, иначе запрос не будет выполнен и вернется ошибка. Если значение атрибута пустое (ISQLType::IsEmpty), то значение не будет вставлено в таблицу значений (если его не было), или будет удалено из таблицы значений (если оно было). (Смысл EAV в том, что мы не храним null) Вне зависимости от того, пришла пустая или непустая (ISQLType::IsEmpty) переменная value, атрибут будет добавлен в таблицу атрибутов (если его там ещё не было).
